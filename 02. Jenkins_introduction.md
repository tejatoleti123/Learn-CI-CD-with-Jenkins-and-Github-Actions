# Jenkins – Introduction and Architecture

---

## What is Jenkins?

Jenkins is an **open-source automation server** used to build CI/CD pipelines.

It helps automate:
- Code builds
- Testing
- Packaging
- Deployment

---

## Why Jenkins is widely used

- Free and open-source
- Large plugin ecosystem
- Supports any language or platform
- Highly customizable
- Widely adopted in enterprises

---

## Jenkins Architecture

Jenkins follows a **Controller–Agent architecture**.

### Jenkins Controller
- Provides Web UI
- Manages jobs and pipelines
- Schedules builds
- Distributes work to agents

### Jenkins Agent
- Executes the actual jobs
- Can run on Linux or Windows
- Can be physical, virtual, or container-based

---

## Jenkins in real-world environments

Typical usage:
- Jenkins Controller on a server or VM
- Agents on separate machines
- Jenkins integrated with Git repositories
- Jenkins integrated with cloud platforms

---

## Why Jenkins is still relevant

Even with newer tools:
- Jenkins supports complex workflows
- Jenkins works with any SCM
- Jenkins is extremely flexible

Jenkins remains a strong choice when customization is required.
