# Jenkins Freestyle Jobs

## Overview

**Jenkins Freestyle jobs** are the simplest type of job in Jenkins. They are primarily **UI-driven** and help beginners understand how Jenkins executes tasks step by step.

Freestyle jobs are ideal for learning:
- How Jenkins pulls source code
- How build steps are executed
- How job configuration flows inside Jenkins

This is an important stepping stone before moving to pipelines.

---

## What is a Jenkins Freestyle Job?

A Freestyle job is a:
- Manually configured Jenkins job
- Defined using the Jenkins Web UI
- Sequence of steps executed on an agent

Each job typically includes:
- Source code configuration
- One or more build steps
- Optional post-build actions

# Jenkins Pipelines

## Overview

**Jenkins Pipelines** are the modern and recommended way to define CI/CD workflows.

Unlike Freestyle jobs, pipelines are defined as **code**, usually in a file called `Jenkinsfile`.

This approach is known as **Pipeline as Code**.

---

## What is a Jenkins Pipeline?

A Jenkins Pipeline is:
- A set of steps written as code
- Stored alongside application source code
- Executed automatically by Jenkins

Pipelines enable:
- Version control of CI/CD logic
- Reusability
- Better collaboration
- Consistent execution across environments

---

## Pipeline as Code (Why It Matters)

Defining pipelines as code allows you to:

- Track changes using Git
- Review CI/CD logic like application code
- Reproduce pipelines easily
- Avoid manual UI-based configuration drift

This is the industry-standard approach.

---

## Sample Pipeline Flow (Conceptual)

A typical CI pipeline follows this sequence:

1. Checkout source code
2. Build the application
3. Run automated tests
4. Archive build artifacts

Each stage runs only if the previous stage succeeds.

---

## Why Pipelines Are Preferred

Pipelines are preferred because they:

- Treat CI/CD as code
- Improve collaboration
- Scale better than Freestyle jobs
- Align with modern DevOps practices

---

## Key Takeaway

- Freestyle jobs help you understand Jenkins basics
- Pipelines are the recommended long-term approach
- Learning both builds strong Jenkins fundamentals
